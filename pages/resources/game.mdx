# Games

Games.

### Game Object

This structure is a superset of the [detectable application](/resources/application#detectable-application-object) object with the following additional fields:

| Field
| companies               | array[[company](#company-structure) object]           | The companies working on a game                     |
| supplemental_game_data? | [game data](#game-data-structure) object              | The supplemental game data                          |
| genres?                 | array[integer]                                        | The [genres](/resources/store#sku-genre) of the SKU |
| platforms?              | array[integer]                                        | The platforms                                       |
| screenshot_urls?        | array[string]                                         | The URLs to the game screenshots                    |
| websites?               | array[[game website](#game-website-structure) object] | The websites                                        |
| l30_rank?               | integer                                               | ?                                                   |

<RouteHeader method="GET" url="/games/detectable" unauthenticated>
  Get Detectable Games
</RouteHeader>

Returns a list of [detectable application](/resources/application#detectable-application-object) objects representing games that can be detected by Discord for rich presence.

<RouteHeader method="GET" url="/games/detectable/exclusions" unauthenticated>
  Get Detectable Game Exclusions
</RouteHeader>

Returns the patterns which should be ignored when detecting games.

###### Response Body

| Field       | Type          | Description                                                                       |
| ----------- | ------------- | --------------------------------------------------------------------------------- |
| executables | array[string] | Names of the ignored processes                                                    |
| patterns    | array[string] | Regular expression patterns which will be matched against process executable name |

<RouteHeader method="GET" url="/games">
  Get Games
</RouteHeader>

Returns a list of [game](#game-object) objects for the given application IDs.

###### Query String Params

| Field                  | Type             | Description                                                          |
| ---------------------- | ---------------- | -------------------------------------------------------------------- |
| game_ids               | array[snowflake] | The IDs of the applications (1-25)                                   |
| with_supplemental_data | boolean          | Whether to include the `supplemental_game_data` field (default true) |

<RouteHeader method="GET" url="/games/{application.id}/updates/{update_id}">
  Get Game Update
</RouteHeader>

The update ID must be 20 or fewer in the length.
