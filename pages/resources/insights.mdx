# Insights

Insights are a data monitoring and analytics tool that Discord provides to help community managers understand their server's performance and how their members are interacting with it.

#### Interval Values

| Value | Description |
|-------|-------------|
| 1     | Daily       |
| 2     | Weekly      |
| 3     | Monthly     |

## Endpoints

<RouteHeader method="GET" url="/guilds/{guild.id}/analytics/growth-activation/overview">
  Get Overview
</RouteHeader>

###### Query String Params
| Field          | Type         | Description                                      |
| -------------- | ------------ | ------------------------------------------------ |
| start          | ISO8601 date | The start date for the insights data             |
| end            | ISO8601 date | The end date for the insights data               |
| [interval](#interval-values) | integer | The interval for the insights data |

###### Response Body

| Field             | Type         | Description                                      |
| ----------------- | ------------ | ------------------------------------------------ |
| day_pt            | ISO8601 date | The date of the data point                       |
| new_members       | integer      | The number of new members who joined the server  |
| new_communicators | integer      | The number of new members who communicated in the server |

<RouteHeader method="GET" url="/guilds/{guild.id}/analytics/growth-activation/joins-by-referrer">
  Get Joins by Referrer
</RouteHeader>

###### Query String Params

| Field          | Type         | Description                                      |
| -------------- | ------------ | ------------------------------------------------ |
| start          | ISO8601 date | The start date for the insights data         |
| end            | ISO8601 date | The end date for the insights data           |
| [interval](#interval-values) | integer | The interval for the insights data |

###### Response Body

| Field               | Type         | Description                                      |
| ------------------- | ------------ | ------------------------------------------------ |
| day_pt              | ISO8601 date | The date the referral was made                   |
| referring_domain^1^ | string       | The referring domain that led to the referral link being clicked |
| joins               | integer      | The number of users who joined the server using this referral link |

^1^ The `referring_domain` field represents what the `Referer` header contained when they joined the server.

<RouteHeader method="GET" url="/guilds/{guild.id}/analytics/growth-activation/joins-by-source">
